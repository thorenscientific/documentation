VCU118 Quick start
===============================================================================

.. figure:: ../images/ad9213_vcu118_ports.jpeg
   :alt: VCU118 board with labeled JTAG, UART, Ethernet, and FMC+ port locations
   :width: 800

   VCU118 port locations

.. esd-warning::

This guide provides quick instructions on how to set up the
:adi:`EVAL-AD9213` on:

- :xilinx:`VCU118` on the FMC+ connector

Using Linux as software
-------------------------------------------------------------------------------

Necessary files
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

The following files are needed to program the system:

- HDL bitstream: ``system_top.bit``, built from the
  :external+hdl:ref:`Build an HDL project <build_hdl>`
- Linux kernel image: ``simpleImage.vcu118_ad9213_evb.strip``,
  built from the :ref:`linux-kernel microblaze`
- TCL script:

.. collapsible:: run.tcl

   .. code-block:: tcl

      connect
      fpga -f system_top.bit
      after 1000
      target 3
      dow simpleImage.vcu118_ad9213_evb.strip
      after 1000
      con
      disconnect

Gather all three files into a single working directory on the host PC
before proceeding.

.. note::

   Pre-built files for this reference design are not yet available.
   The files must be built manually using the links above. Official release
   artifacts will be provided here once available. You can also check the
   in-development Linux branch at :git-linux:`9213 branch <9213:/>` and the
   HDL repository at :git-hdl:`ad9213_evb <projects/ad9213_evb>`.

Required software
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

- Xilinx ``xsct`` or ``xsdb`` (included in Vivado/Vitis toolchain)
- A UART terminal (e.g. PuTTY/Tera Term/Minicom) at 115200 baud rate (8N1)

Required hardware
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

- AMD Xilinx :xilinx:`VCU118` FPGA board and its power supply
- :adi:`EVAL-AD9213` evaluation board
- ZX180V-HSPC FMC VITA 57.4 Mezzanine card breakout adapter
- 2x Micro-USB cables (JTAG and UART)

More details as to why you need these can be found at
:ref:`ad9213 evb prerequisites`.

.. _ad9213 evb quickstart vcu118 hw-changes:

.. warning::

   The following modification is required before powering on the system:

   The :adi:`EVAL-AD9213` does not support JTAG boot in its
   default configuration. To enable it, insert the evaluation board into
   the ZX180V-HSPC FMC+ breakout adapter and solder a wire between
   pins D30 and D31 on the breakout adapter. This connection
   bridges the TDI and TDO lines, which is necessary to close the JTAG
   chain. See more details below.

Testing
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Creating the setup
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

.. figure:: ../images/ad9213_vcu118_setup.jpeg
   :alt: VCU118 FPGA board with EVAL-AD9213 mounted via FMC+ breakout adapter
   :width: 800

   VCU118 with EVAL-AD9213 hardware setup

Follow the steps below in order to avoid damaging the components:

#. Solder a wire between pins D30 and D31 on the ZX180V-HSPC
   breakout adapter

   .. figure:: ../images/ZX180V_HSPC_connection.jpeg
      :alt: ZX180V-HSPC breakout adapter showing the D30 and D31 pins to be bridged
      :width: 400

      ZX180V-HSPC breakout adapter with TDI/TDO pins labeled

#. Insert the :adi:`EVAL-AD9213` evaluation board into the
   ZX180V-HSPC FMC+ breakout adapter
#. Insert the FMC+ breakout adapter into the FMC+ connector of the
   :xilinx:`VCU118`
#. Locate SW16 on the VCU118, which is the JTAG boot mode switch:

   .. figure:: ../images/ad9213_vcu118_jtag.jpeg
      :alt: SW16 DIP switch positions on the VCU118 for JTAG boot mode
      :width: 400

      VCU118 SW16 switch configured for JTAG boot mode

#. Configure switch SW16 for JTAG boot mode:

   .. list-table::
      :header-rows: 1

      * - SCE (SW16-1)
        - M2 (SW16-2)
        - M1 (SW16-3)
        - M0 (SW16-4)
      * - OFF
        - ON
        - OFF
        - ON

#. Connect the first Micro-USB cable to the JTAG port on the VCU118
#. Connect the second Micro-USB cable to the UART port on the VCU118
#. Plug-in an Ethernet cable from your router/switch to the Ethernet port on
   the FPGA board
#. Connect the external power supply to the evaluation board
#. Power on the VCU118
#. Observe serial console output messages on your terminal program.

Programming the board
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

With the board powered on, open ``xsct`` or ``xsdb`` on the host PC,
navigate to the working directory containing the required files, and
source the programming script:

.. shell::

   $xsct
    xsct% source run.tcl

The script will program the FPGA with the bitstream and load the Linux
kernel image over JTAG. It will build the MicroBlaze system and start the boot
process. This may take a few minutes to complete. Do not interrupt the process
once it has started. Build logs will be printed in the terminal.

Useful commands
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

The below commands are to be run in the serial terminal connected to the FPGA.

To find out the IP of the FPGA board, run the following command and take the
IP specified at "eth0 inet":

.. shell::

   $ifconfig


To find the IIO devices detected, run:

.. shell::

   $iio_info | grep iio:device

To power off the system cleanly, run the following command and wait for
the final message before switching off the board:

.. shell::

   $poweroff

.. important::

   This system boots from a ramdisk. It is a MicroBlaze soft processor.
   The filesystem is not persistent.
   Any changes made at runtime will be lost. At the same time, the
   system cannot reboot, so the only way to restart the system is to
   reprogram the FPGA and reload the kernel image using the steps above.

IIO Oscilloscope
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

:ref:`iio-oscilloscope` is a cross-platform is a cross-platform
GUI application that can connect to the board remotely over the network and
capture data from IIO devices.

.. note::

   A signal generator is required to provide an analog input signal to the
   evaluation board's RF SMA input connector before capturing data.
   The captures below were obtained with a signal generator set to:
   20 MHz, 1.5 V\ :sub:`pp`\. The time domain
   capture uses a sample size of 4096 samples. The frequency domain
   capture uses an FFT size of 65536.

To connect, open IIO Oscilloscope on the host PC, go to
Setting, Connect, check Manual option and enter the URI using the board
IP address obtained from ``ifconfig``:

.. code-block:: none

   ip:<board-ip>

Press Refresh to list the available IIO devices (FRU Info, Context
Description, Context Attributes and IIO Devices fields should display
information about the connected devices), then press Connect.

.. figure:: ../images/ad9213_iio_connect.jpeg
   :alt: IIO Oscilloscope connection dialog with manual IP entry selected
   :width: 700

   IIO Oscilloscope connect window

Once connected, select the AD9213 device and use the Capture window to
acquire and display ADC samples.

.. figure:: ../images/adi9213_iio_time.jpeg
   :alt: IIO Oscilloscope showing AD9213 time domain waveform
   :width: 700

   AD9213 time domain capture in IIO Oscilloscope

   :alt: IIO Oscilloscope showing AD9213 frequency domain FFT capture
   :width: 700
